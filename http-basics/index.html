HTTP Basics

Terms

  • HTTP Hyper Text Transfer Protocol - a protocol or language which determines how browsers and servers communicate
  • DNS Domain Name Service
  • Request body Usually blank, but can contain data you are sending to the server.
  • Request headers The portion of the request that determines how the server should respond.
  • Response headers The portion of the response which tells the client how the server responded.
  • Response Body If successful, the content of the requested resource
  • HTTP Status Code numerical code (200, 404, 403) which tells the client the status of the response.
  • IP Address Internet Protocol Address - a number like 65.45.34.123 which is an address for a specific computer or network of computers.
  • Client A device which communicates with a server using a protocol such as HTTP
  • Server A specializedz computer which in the purpose of HTML, responds to client requests.
  • Domain Name A name like "google.com" which resolves to an IP address using DNS
  • Protocol A language which determines how computers can speak to each other.
  • Parse Interpret meaning from data given a specific grammer. A browser will parse HTML to display a web-site.

Topics

    1. HTTP is a protocol (language) that computers can speak with each other.

      It consists basically of a client request message: 

       client: "Hi"

      and a server response message: 

       server: "Hello!"
      Request

      A request consists of request headers , which include request method, the resource location, the protocol, the host, and sometimes a request body .

      An actual HTTP request might look something like this: 

      GET /index.html HTTP/1.1
Host: www.example.com

      Where GET is the method, /index.html is the url of the resource, HTTP/1.1 is the protocol to use, and Host: www.example.com is the server to send the request to.

      (There is no request body for this example)

      An example of a POST request with a request body: 

      POST /login.php HTTP/1.1
Host: www.example.com

name=joe&password=test123

      Where POST is the method, /login.php is the url of the resource, HTTP/1.1 is the protocol to use, Host: www.example.com is the server to send the request to, and name=joe@password=test123 is the request body , which is seperated from the request headers by a blank line.

      In summary, these are called the request headers.

      There are additional headers you can specify, like requested language and content-type.

      The request method GET is one of several HTTP methods or HTTP verbs.

      These verbs define how the server should respond to the request.

      The important methods are:

      GET 'gets' the requested resource. This is the majority of requests.

      POST 'posts' information to the requested resource. This is uploads, logins, registrations, posts, comments etc. Generally whenever you are sending information to the server.

      Other commonly used methods: 

       HEAD, PUT, DELETE

      Other methods you probably won't need to know: 

       OPTIONS, TRACE, CONNECT

      The request message is sent to the server, and will hopefully result in a response.

      Response

      A server response consists of response headers and usually a response body .

      A typical response will look like this: 

      HTTP/1.1 200 OK
Date: Mon, 23 May 2005 22:38:34 GMT
Server: Apache/1.3.3.7 (Unix) (Red-Hat/Linux)
Last-Modified: Wed, 08 Jan 2003 23:11:55 GMT
Etag: "3f80f-1b6-3e1cb03b"
Accept-Ranges: bytes
Content-Length: 438
Connection: close
Content-Type: text/html; charset=UTF-8

<!DOCTYPE html>...

      With the above being the response headers , and the response body, <!DOCTYPE html>, begins after a blank line below the headers. This will be the content of the resource you requested (some HTML, an image, etc) depending on request method and resource availablity.

      An important thing to learn from this is this part: 

       200 OK

      This is an HTTP Status Code

      HTTP status codes tell us briefly what the result of the request was.

      The important codes and their definitions are:

      • 200 - OK request was completed successfully
      • 301 - request is permanently redirected to somewhere else
      • 302 - request is temporarily redirected to somewhere else
      • 304 - response has not changed
      • 403 - forbidden from accessing the resource
      • 404 - requested resource was not found
      • 500 - some internal error completing the request
      • more

      The requesting software will then take the response and do something with it, such as generate a web-page from the HTML in the response, or play a video.

    1. Briefly, the client/server relationship is defined by

      A client which can be any device capable of communicating in HTTP , or any of the other protocols.

      And a server which can be any device or devices capable of responding to client communications.

      For the purposes of websites, this relationship generally refers to a client being a web-browser of some sort (could be on a computer, a phone, whatever), and a web-server which theoretically could also be any device, but will most like be a special type of computer called a server (or several servers).

    1. As described above, the web-browser / web-server relationship is a specific type of client / server relationship.

      In general, a web-browser will communicate with a web-server using HTTP

      The browser will both make the request and parse (display) the response.

      How the browser knows which server to contact is by a system called DNS

      This basically transforms a domain-name like "google.com" into an IP-Address like 65.45.34.123

      Your browser only sends the domain-name portion of the request in HTTP (like above) - the DNS part is done after the request leaves your browser.

      After the appropriate server is contacted, it generates an HTTP response and sends it back to your browser

      Your browser receives as a HTTP response and then chooses what to do with it.

      In a case where you are downloading a web-page, the response-body will consist of HTML, so your browser will parse and display that.
